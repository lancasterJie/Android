# Android ListView作业：待办事项清单应用实验报告

## 一、实验目的
通过本实验，掌握Android开发中ListView控件的使用和任务管理应用的基本实现：

1. **ListView控件**：掌握ListView作为列表容器的基本使用方法
2. **自定义适配器**：学会创建自定义ArrayAdapter或BaseAdapter
3. **数据模型设计**：掌握Java类的设计与封装
4. **用户界面布局**：学习Android界面布局的设计与实现
5. **基本交互功能**：实现添加、删除、标记完成等基本任务管理功能

## 二、实验内容与功能实现

### 2.1 添加任务功能

#### 2.1.1 功能描述
通过对话框实现新任务的添加，用户可以输入任务名称和截止日期。

#### 2.1.2 技术实现
1. **对话框设计**：使用AlertDialog创建输入对话框
2. **布局文件**：dialog_add_task.xml定义输入界面
3. **数据验证**：验证输入内容是否为空
4. **数据添加**：将新任务添加到任务列表并刷新显示

#### 2.1.3 实现效果
- 点击"添加新任务"按钮弹出对话框
- 输入任务名称和截止日期
- 点击"添加"按钮将任务加入列表
- 添加成功后有Toast提示

### 2.2 删除任务功能

#### 2.2.1 功能描述
提供两种删除任务的方式：长按任务项弹出确认对话框或点击任务项的删除按钮。

#### 2.2.2 技术实现
1. **长按事件**：ListView的OnItemLongClickListener
2. **删除按钮**：列表项中的删除按钮点击事件
3. **确认对话框**：删除前显示确认对话框防止误操作
4. **数据移除**：从ArrayList中移除任务并刷新列表

#### 2.2.3 实现效果

- 点击任务项右侧的✕按钮 → 弹出确认对话框 → 确认删除
- 删除后列表自动更新，显示剩余任务

### 2.3 标记任务完成功能

#### 2.3.1 功能描述
通过复选框标记任务的完成状态，已完成任务有视觉区分。

#### 2.3.2 技术实现
1. **复选框控件**：每个任务项包含一个CheckBox
2. **状态监听**：setOnCheckedChangeListener监听状态变化
3. **视觉反馈**：根据完成状态设置文字颜色和删除线
4. **数据更新**：更新TaskItem的isCompleted属性

#### 2.3.3 实现效果
- 点击任务前的复选框切换完成状态
- 已完成任务：灰色文字 + 删除线效果
- 未完成任务：黑色文字 + 正常显示
- 状态变化后自动更新底部统计信息
